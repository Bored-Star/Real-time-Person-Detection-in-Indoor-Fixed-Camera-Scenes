# 系统架构说明文档

## 系统概述

本系统是一个端云协同的视频流人脸检测解决方案，采用两设备架构：

- **设备A（端侧）**：部署在计算资源有限的设备上，负责视频采集和预处理
- **设备B（边缘侧）**：部署在计算能力较强的边缘服务器上，负责AI推理和业务逻辑

## 模块职责详解

### 设备A模块

#### 1. 视频采集模块 (`capture.py`)

**职责**：
- 从视频文件读取帧
- 支持帧率控制（降采样）
- 处理视频结束和异常

**关键类**：
- `VideoCapture`：视频采集器主类

**输入**：视频文件路径
**输出**：原始帧（numpy数组，BGR格式）

**设计决策**：
- 使用OpenCV的`VideoCapture`读取视频
- 支持with语句确保资源释放
- 生成器模式提供逐帧读取

---

#### 2. 帧预处理模块 (`preprocess.py`)

**职责**：
- 调整帧分辨率
- JPEG压缩
- 平衡画质和传输效率

**关键类**：
- `FramePreprocessor`：帧预处理器主类

**输入**：原始帧（numpy数组）
**输出**：JPEG压缩的字节数据

**设计决策**：
- 保持宽高比的缩放
- 可配置JPEG质量参数
- 支持自定义目标尺寸

---

#### 3. TCP发送模块 (`sender.py`)

**职责**：
- 建立TCP连接
- 发送压缩帧数据
- 断线重连机制

**关键类**：
- `FrameSender`：帧发送器主类

**输入**：JPEG字节数据
**输出**：网络传输流

**设计决策**：
- 采用长度前缀分帧协议（4字节大端长度 + 数据）
- 自动重连机制（指数退避）
- 支持with语句管理连接生命周期

---

### 设备B模块

#### 1. TCP接收模块 (`receiver.py`)

**职责**：
- 监听TCP连接
- 接收帧数据
- 解析长度前缀分帧协议

**关键类**：
- `FrameReceiver`：帧接收器主类

**输入**：TCP网络流
**输出**：JPEG字节数据

**设计决策**：
- 服务端模式，监听指定端口
- 生成器模式提供连续帧流
- 处理连接断开和超时

---

#### 2. AI推理模块 (`detector.py`)

**职责**：
- 加载OpenCV DNN模型
- 执行人脸检测
- 输出标准化检测结果

**关键类**：
- `FaceDetector`：人脸检测器主类
- `Detection`：检测结果数据类

**输入**：图像帧（numpy数组或JPEG字节数据）
**输出**：检测结果列表（边界框、置信度）

**设计决策**：
- 支持多种模型格式（ONNX、Caffe等）
- 置信度阈值过滤
- 非极大值抑制（NMS）

---

#### 3. 跟踪业务模块 (`tracker.py`)

**职责**：
- 帧间目标匹配
- 人流计数
- 异常停留检测和报警

**关键类**：
- `ObjectTracker`：目标跟踪器主类
- `Track`：跟踪目标数据类

**输入**：检测结果序列
**输出**：跟踪结果、统计信息、报警事件

**设计决策**：
- 使用IoU（交并比）进行帧间匹配
- 为每个目标分配唯一跟踪ID
- 检测异常停留（指定区域内超过阈值时间）

---

#### 4. 结果输出模块 (`result_writer.py`)

**职责**：
- 在帧上绘制检测和跟踪结果
- 保存标注后的帧
- 输出统计日志
- 记录报警事件

**关键类**：
- `ResultWriter`：结果写入器主类

**输入**：原始帧、检测结果、跟踪结果
**输出**：标注图像、控制台日志、统计文件

**设计决策**：
- 支持可视化（边界框、ID、报警标记）
- 间隔保存减少磁盘占用
- 实时控制台输出统计信息

---

## 数据流详解

### 端到端数据流

1. **设备A侧**：
   ```
   视频文件 → VideoCapture → 原始帧
   → FramePreprocessor → JPEG字节数据
   → FrameSender → TCP网络流
   ```

2. **网络传输**：
   ```
   TCP Socket（长度前缀分帧）
   格式: [4字节长度][JPEG数据]
   ```

3. **设备B侧**：
   ```
   TCP网络流 → FrameReceiver → JPEG字节数据
   → FaceDetector → 检测结果
   → ObjectTracker → 跟踪结果+报警
   → ResultWriter → 标注帧+日志
   ```

## 通信协议

### 长度前缀分帧协议

每个数据包由两部分组成：

```
┌──────────────┬──────────────────┐
│  Length (4B) │  Frame Data      │
│  Big-Endian  │  JPEG Bytes      │
└──────────────┴──────────────────┘
```

- **Length**：4字节大端无符号整数，表示Frame Data的字节长度
- **Frame Data**：JPEG压缩的图像数据

**示例代码**：

发送端（Python）：
```python
import struct
frame_bytes = ...  # JPEG压缩的帧
length_prefix = struct.pack('>I', len(frame_bytes))
packet = length_prefix + frame_bytes
sock.sendall(packet)
```

接收端（Python）：
```python
import struct
length_bytes = recv_n_bytes(sock, 4)
length = struct.unpack('>I', length_bytes)[0]
frame_data = recv_n_bytes(sock, length)
```

## 关键设计决策

### 1. 为什么选择长度前缀分帧？

- **简单可靠**：易于实现，边界明确
- **高效**：无需额外转义字符
- **通用**：支持任意大小的帧数据

### 2. 为什么使用JPEG压缩？

- **压缩率高**：相比PNG、BMP节省大量带宽
- **硬件加速**：多数设备有JPEG编解码硬件支持
- **质量可控**：通过质量参数平衡画质和大小

### 3. 为什么使用IoU进行跟踪匹配？

- **平移不变**：对目标移动鲁棒
- **计算高效**：相比深度学习跟踪方法更快
- **易于实现**：OpenCV提供IoU计算函数

### 4. 为什么分离检测和跟踪？

- **模块解耦**：检测和跟踪可以独立优化
- **灵活扩展**：可替换检测模型而不影响跟踪逻辑
- **性能调优**：检测频率可低于跟踪频率

## 扩展性设计

### 支持多种检测模型

系统支持OpenCV DNN兼容的任意模型：
- YuNet（推荐）
- ResNet SSD
- MobileNet SSD
- 自定义ONNX模型

### 支持多客户端

设备B的接收模块可扩展为：
- 为每个客户端创建独立的跟踪器
- 支持多个视频流同时处理

### 支持分布式部署

可扩展为：
- 多个设备A连接到一个设备B
- 设备B集群部署，负载均衡

## 性能指标

### 设备A性能

- **分辨率调整**：640x480 或 320x240
- **压缩质量**：70-85（推荐85）
- **发送帧率**：5-15 FPS
- **网络带宽**：100-500 KB/s（取决于分辨率和压缩质量）

### 设备B性能

- **检测模型**：YuNet（推荐）速度最快
- **推理帧率**：15-30 FPS（取决于硬件）
- **跟踪延迟**：< 100ms
- **内存占用**：~200MB（不含模型）

## 部署场景

### 场景1：本地网络

```
设备A（端侧）───────局域网───────> 设备B（边缘侧）
   IP: 192.168.1.100              IP: 192.168.1.200
```

### 场景2：跨网络

```
设备A（端侧）───────互联网───────> 设备B（云端）
   公网IP                        公网IP + 端口映射
```

### 场景3：多对一

```
设备A1 ──┐
设备A2 ──┼──> 交换机/路由器 ───> 设备B（边缘服务器）
设备A3 ──┘
```

## 安全考虑

1. **网络安全**：
   - 建议使用VPN或加密通道
   - 可选TLS/SSL加密

2. **数据隐私**：
   - 视频帧在内存中处理，不落盘
   - 仅输出结果和标注帧

3. **访问控制**：
   - 可添加IP白名单
   - 可选认证机制

---

## 故障恢复

### 网络中断

- 设备A自动重连（指数退避）
- 设备B等待新连接

### 视频结束

- 设备A正常退出
- 设备B等待新连接或处理其他客户端

### 模型加载失败

- 启动时检查模型文件
- 提供详细错误信息
