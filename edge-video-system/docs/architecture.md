# 系统架构文档

## 概述

本系统是一个端云协同的视频流人脸检测解决方案，将计算任务分布在不同能力的设备上：
- **设备A（端侧）**：负责视频采集和轻量级预处理
- **设备B（边缘侧）**：负责AI推理和业务逻辑处理

## 模块职责

### 设备A模块（端侧）

#### 1. `capture.py` - 视频采集模块
- **职责**：从视频文件读取帧数据，支持帧采样控制
- **输入**：视频文件路径、目标采样帧率
- **输出**：采样的图像帧（numpy数组）
- **关键决策**：
  - 使用OpenCV VideoCapture读取视频
  - 支持可配置的帧采样率以减少传输量

#### 2. `preprocess.py` - 帧预处理模块
- **职责**：图像缩放和JPEG压缩
- **输入**：原始图像帧、目标尺寸、压缩质量
- **输出**：JPEG压缩的字节数据
- **关键决策**：
  - 统一缩放到固定尺寸便于推理
  - JPEG压缩减少网络传输量
  - 保持适当的宽高比

#### 3. `sender.py` - TCP发送模块
- **职责**：建立TCP连接，发送压缩帧数据，处理断线重连
- **输入**：设备B的IP和端口、JPEG帧数据
- **输出**：发送状态（成功/失败）
- **关键决策**：
  - 使用长度前缀协议：4字节大端整数 + 数据体
  - 指数退避的断线重连策略
  - 非阻塞发送避免阻塞主流程

#### 4. `main.py` - 设备A入口
- **职责**：命令行参数解析，协调各模块工作
- **工作流程**：解析参数 → 初始化采集器 → 连接设备B → 逐帧处理循环

### 设备B模块（边缘侧）

#### 1. `receiver.py` - TCP接收模块
- **职责**：监听TCP端口，接收并解析帧数据
- **输入**：监听地址和端口
- **输出**：JPEG帧字节数据
- **关键决策**：
  - 长度前缀协议解析帧边界
  - 精确字节接收避免数据丢失
  - 处理连接断开和超时

#### 2. `detector.py` - AI推理模块
- **职责**：加载OpenCV DNN模型，执行人脸检测
- **输入**：JPEG帧字节数据、模型文件路径
- **输出**：检测结果（边界框+置信度）+ 解码图像
- **关键决策**：
  - 使用OpenCV DNN模块支持多种模型格式
  - CPU推理（边缘侧通常有足够算力）
  - NMS后处理去除重复检测

#### 3. `tracker.py` - 业务逻辑模块
- **职责**：目标跟踪、人流统计、异常停留检测
- **输入**：检测结果列表、帧序号
- **输出**：跟踪对象列表、报警信息
- **关键决策**：
  - 基于IoU的简单匹配算法
  - 停留时间阈值判断异常
  - 内存存储跟踪状态（无持久化）

#### 4. `result_writer.py` - 结果输出模块
- **职责**：保存标注帧、打印统计日志、输出报警
- **输入**：标注图像、跟踪对象、报警列表
- **输出**：图像文件、控制台日志
- **关键决策**：
  - 按帧序号命名保存图像
  - 定期输出统计摘要
  - 报警信息立即高亮显示

#### 5. `main.py` - 设备B入口
- **职责**：命令行参数解析，协调各模块工作
- **工作流程**：解析参数 → 启动接收器 → 加载模型 → 初始化跟踪器 → 处理循环

## 数据流

```
视频文件 → [设备A: capture] → 原始帧
  → [设备A: preprocess] → JPEG字节流
  → [设备A: sender] → TCP网络传输
  → [设备B: receiver] → JPEG字节流
  → [设备B: detector] → 检测结果 + 解码帧
  → [设备B: tracker] → 跟踪对象 + 报警
  → [deviceB: result_writer] → 标注帧 + 日志
```

## 通信协议

### TCP分帧协议
```
+-------------------+----------------------+
| 长度前缀 (4 bytes) | 帧数据 (N bytes)     |
+-------------------+----------------------+
```
- **长度前缀**：大端无符号整数，表示后续帧数据的字节数
- **帧数据**：JPEG压缩的图像字节数据

### 设计考虑
- 简单可靠的分帧方式，避免粘包问题
- 支持4GB以内的帧数据（理论上限）
- 易于实现和调试

## 性能优化点

### 设备A（端侧）
1. **帧采样**：降低传输和计算量
2. **JPEG压缩**：平衡图像质量和传输速度
3. **尺寸缩放**：统一输入尺寸，减少推理时间

### 设备B（边缘侧）
1. **置信度阈值**：减少误检和后续处理
2. **NMS处理**：去除重复检测
3. **IoU匹配优化**：快速跟踪算法
4. **内存管理**：及时释放离开场景的目标

## 部署架构

### 单机部署（测试环境）
```
同一台机器的两个终端：
终端1: python -m device_b.main --host 127.0.0.1 --port 5000
终端2: python -m device_a.main --video test.mp4 --host 127.0.0.1 --port 5000
```

### 局域网部署（生产环境）
```
设备A (192.168.1.10): 端侧设备
设备B (192.168.1.20): 边缘服务器

设备B: python -m device_b.main --host 0.0.0.0 --port 5000
设备A: python -m device_a.main --video test.mp4 --host 192.168.1.20 --port 5000
```

### 广域网部署（云端推理）
```
设备A: 任意网络位置的端侧设备
设备B: 云服务器（公网IP）

需要考虑：网络延迟、数据加密、认证机制
```

## 监控与运维

### 关键指标
- 端到端延迟（帧采集 → 检测结果）
- 网络传输速率和丢包率
- 检测准确率和召回率
- 跟踪准确度
- 异常报警准确率

### 日志级别
- DEBUG: 详细的数据流信息
- INFO: 统计摘要和关键事件
- WARNING: 网络问题和模型性能
- ERROR: 严重故障和异常

### 故障处理
- 网络断开：自动重连（指数退避）
- 模型加载失败：降级到备用模型
- 内存不足：降低处理质量或扩容
- 检测失败：跳过当前帧，继续处理
